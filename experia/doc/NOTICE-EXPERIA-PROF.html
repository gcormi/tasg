1 Qu'est-ce qu'Experia ?

Experia est une application web éducative qui permet à un professeur de :

  • Fournir un corpus — texte libre, fichier PDF, TXT ou HTML (cours, programme, article…)
  • Générer automatiquement des activités — l'IA Albert crée les questions à partir de votre contenu, en quelques secondes
  • Partager une session — les élèves accèdent via un lien + un code que vous choisissez vous-même (ex : energies-5e1)
  • Recevoir des résultats auto-corrigés — selon le barème que vous avez défini, sans correction manuelle
Experia fonctionne avec l'IA souveraine Albert (Étalab / État français) et stocke les données sur votre espace Nuage apps.education.fr. Aucune donnée ne quitte les serveurs de l'Éducation Nationale.

2 Prérequis & première connexion

Étape 1 — Créer le dossier Experia sur votre Nuage

Dans l'interface web de votre Nuage, créez un dossier nommé exactement experia (en minuscules) à la racine de vos fichiers. C'est là que seront stockées toutes les sessions et résultats.

Étape 2 — Générer un mot de passe d'application Nuage

  1. Connectez-vous sur votre Nuage (nuage03.apps.education.fr ou nuage02, etc.)
  2. Allez dans Paramètres → Sécurité → Mots de passe d'application
  3. Tapez un nom (ex: Experia) et cliquez Créer
  4. Copiez le mot de passe généré — il ne s'affiche qu'une seule fois !
Utilisez toujours un mot de passe d'application, jamais votre mot de passe ENT principal. Le mot de passe d'application a une portée limitée et peut être révoqué à tout moment.

Étape 3 — Se connecter dans Experia

Sur l'écran de connexion, renseignez simplement :

Identifiant Nuage votre identifiant court ENT (ex: gcormi)
Mot de passe le mot de passe d'application généré à l'étape 2

Les URLs n8n et le serveur Nuage sont pré-remplis avec les bonnes valeurs. Vous ne devriez pas avoir à les modifier (accessibles via le bouton "Paramètres avancés").

Mémorisation automatique : après la première connexion, votre identifiant, mot de passe et clé Albert sont mémorisés dans le navigateur. Vous n'aurez pas à les ressaisir lors des prochaines connexions sur le même PC.
Lien de connexion rapide : sur l'écran de connexion, cliquez "Générer mon lien de connexion rapide" (sous le bouton Se connecter). Copiez et conservez ce lien (Post-it, notes perso). En ouvrant ce lien sur n'importe quel PC, vous êtes connecté automatiquement sans rien saisir. ⚠️ Ce lien contient vos identifiants — gardez-le pour vous.
Ordinateurs partagés : Si vous utilisez l'ordinateur de la salle des profs, cliquez bien sur "Déconnexion" à la fin. Cela effacera instantanément vos identifiants sauvegardés en mémoire !

Étape 4 — Obtenir une clé API Albert

La clé API Albert est nécessaire pour que l'IA génère les questions et corrige les réponses ouvertes. Elle est réservée aux agents de l'Éducation Nationale.

Si vous utilisez déjà Mentoria ou Gamia, vous possédez déjà une clé Albert — c'est la même.

3 Créer une session

Titre de la session

Donnez un titre clair que vos élèves verront sur leur écran (ex: "La vie de Mozart", "Les énergies renouvelables").

Code d'accès élèves

Vous choisissez librement le code que vos élèves saisiront pour rejoindre la session. Exemples :

energies-5e1 mozart-3e2 revisions-brevet

Les espaces sont automatiquement remplacés par des tirets. Le code doit être unique parmi vos sessions.

Le corpus

C'est le contenu à partir duquel l'IA génère toutes les questions. Trois options :

  • Texte libre — coller directement votre cours, vos notes, une description de programme
  • Fichier PDF ou TXT — le texte est extrait automatiquement
  • Fichier HTML — les balises sont supprimées, seul le texte est conservé pour la génération
Plus le corpus est précis et structuré, plus les questions seront pertinentes. Idéalement entre 200 et 3 000 mots.

Configuration des activités

Pour chaque activité, vous pouvez :

  • La cocher ou décocher — les activités décochées ne sont pas générées et n'apparaissent pas chez les élèves
  • Définir le nombre de questions (colonne Q.)
  • Définir le barème en points (colonne pts) — le score de l'élève sera automatiquement converti sur cette base
ActivitéQ. par défautPts par défaut
Quiz QCM1010
Paires à relier88
Mission rapidité88
Compléter la phrase55
Détection d'intrus55
Questions ouvertes IA320

Exemple : QCM sur 10 questions, barème 8 pts → un élève qui réussit 7/10 obtient 5,6/8.

Paramètres pédagogiques

  • Activités rejouables — si coché, l'élève peut recommencer une activité. Décochez pour une évaluation en une seule tentative.
  • Afficher la correction (décoché par défaut, recommandé) — si décoché, l'élève voit uniquement ✓ ou ✗ sans voir la bonne réponse. La correction complète reste disponible en fin de session. Cela force la réflexion et évite le tâtonnement aléatoire.
  • Thème visuel — choisissez la couleur d'ambiance de l'espace élève via les pastilles colorées (Classique, Nature, Océan, Soleil, Crépuscule, Bonbon).
La génération prend 30 à 90 secondes selon le nombre d'activités et la taille du corpus. Une barre de progression indique l'avancement activité par activité.

4 Partager avec les élèves

Une fois la session générée, l'écran affiche le barème complet (total de points par activité) et deux informations à transmettre aux élèves :

Le lien d'accès

À copier via le bouton "Copier". Partagez-le via votre ENT, Pronote, ou en le projetant au tableau.

Le code session

Le code que vous avez choisi (ex: energies-5e1). L'élève le saisit sur la page de connexion.

Côté élève : l'élève saisit son identifiant ENT et le code de session. Il peut reprendre là où il s'était arrêté s'il n'a pas eu le temps de tout terminer — ses scores partiels sont sauvegardés automatiquement après chaque activité.
Télécharger la correction

Sur cet écran de partage, vous avez accès à un bouton "Télécharger la correction". Il vous permet d'enregistrer instantanément un document HTML (imprimable en PDF) contenant toutes les réponses et grilles de votre session pour la préparation de votre cours !

5 Consulter les résultats

Actualisation automatique (30s)

Dès que vous ouvrez une session, les résultats se rechargent automatiquement toutes les 30 secondes. Un indicateur sous le titre affiche l'heure de la dernière mise à jour. Vous pouvez aussi cliquer "Actualiser" manuellement à tout moment.

Travaux rendus vs en cours

Un élève qui clique "Rendre le travail" transmet son devoir définitivement (champ submitted: true dans le JSON). Son accès à la session est verrouillé. Les élèves qui ont cliqué "Continuer plus tard" restent en état partiel et peuvent revenir la semaine suivante avec le même lien.

Résultats partiels

Un élève qui n'a pas terminé apparaît avec un badge "X/Y activités" et une icône horloge. Ses scores sont sauvegardés automatiquement au fil de l'avancement.

Score en points réels

Le score de l'élève est affiché en X / Y pts où Y est le total du barème de toutes les activités de la session. Les activités non faites comptent 0 pt et apparaissent en grisé.

Note et appréciation (optionnel)

Vous pouvez ajouter une note /20 et un commentaire personnel. Ces informations sont sauvegardées sur votre Nuage.

6 Gérer ses sessions

Dupliquer une session (même contenu, autre classe)

Sur l'écran d'une session, cliquez l'icône Dupliquer (à gauche de la corbeille). Une boîte vous demande :

  1. Le nouveau titre (ex: "Les énergies — 5e2")
  2. Le nouveau code d'accès (ex: energies-5e2)

La session est créée instantanément — Albert n'est pas rappelé, les mêmes questions sont réutilisées. Chaque classe aura ses propres résultats séparés.

Supprimer une session

Deux façons de supprimer :

  • Depuis la liste — survolez une session, l'icône corbeille apparaît à droite. Cliquez pour supprimer sans ouvrir la session.
  • Depuis la session — bouton corbeille en haut à droite de l'écran session.
La suppression efface le fichier session ET tous les résultats élèves associés du Nuage. Une confirmation affiche le nombre de résultats qui seront supprimés. Action irréversible.

7 Les 8 activités auto-corrigées

Quiz QCM — Questions à 4 choix, correction immédiate avec explication.

Correction : comparaison à la bonne réponse (index).

Paires à relier — Associer termes et définitions en cliquant. Toute mauvaise association est flashée en rouge et réinitialisée — l'élève peut réessayer mais perd des points (−0,5 pt par erreur).

Correction : vérification des paires + pénalité pour les erreurs.

Mots mêlés — Grille générée dynamiquement où l'élève doit retrouver les mots cachés (à la verticale, à l'horizontale, etc.). Sélection par glissement.

Correction : vérification automatique du tracé.

Mots croisés — Grille interactive croisée calculée automatiquement à partir des mots et définitions. Frappe fluide au clavier.

Correction : dès qu'un mot est complet et correct, il se valide en vert.

Mission rapidité — QCM avec chrono 10s par question. Plus l'élève répond vite, plus il marque de points.

Correction : QCM + bonus proportionnel au temps restant.

Compléter la phrase — Trouver le mot manquant. Les fautes légères de frappe sont tolérées.

Correction : distance de Levenshtein avec 20% de tolérance sur la longueur du mot.

Détection d'intrus — Parmi 4 affirmations, repérer la seule qui est fausse.

Correction : comparaison à l'indice de la fausse affirmation.

Questions ouvertes IA — Réponse libre développée, notée par Albert sur 10 avec un commentaire personnalisé. Les réponses ne sont pas auto-corrigées en temps réel : Albert évalue chaque réponse au moment où l'élève clique "Faire évaluer".

Correction : évaluation par l'IA Albert via le corpus de référence (score 0–10 + feedback).

8 Partager l'appli avec un collègue

Un collègue souhaitant utiliser Experia aura besoin de :

1
Son identifiant et mot de passe d'application Nuage — créé sur son propre espace Nuage (nuage01/02/03… selon son académie)
2
Un dossier experia créé sur son Nuage — à la racine, en minuscules, avant la première utilisation
3
Une clé API Albert — réservée aux agents EN. Si votre collègue utilise déjà Mentoria ou Gamia, il possède déjà cette clé.
Les URLs n8n sont partagées et pré-remplies — votre collègue n'a pas à les modifier. Chaque professeur a ses propres sessions stockées sur son Nuage personnel, totalement indépendant.

9 Confidentialité & RGPD

Application conforme au RGPD

Experia respecte pleinement le Règlement Général sur la Protection des Données (RGPD — Règlement UE 2016/679) et les recommandations de la CNIL pour les outils numériques scolaires.

Aucune collecte de données personnelles par l'application

Experia ne dispose d'aucun serveur propre, d'aucune base de données tierce, d'aucun compte utilisateur. Il n'existe aucun point où des données sont collectées en dehors de votre propre espace Nuage.

Stockage exclusivement sur le Nuage apps.education.fr

Toutes les données (sessions, résultats élèves) sont stockées uniquement dans votre espace Nuage personnel, hébergé par le Ministère de l'Éducation Nationale sur des serveurs français. Vous êtes l'unique responsable de traitement de ces données.

Données élèves minimisées (principe de minimisation)

Les élèves ne saisissent que leur identifiant ENT (que vous définissez — prénom seul, initiales, code de classe...). Aucun e-mail, aucun mot de passe élève, aucune donnée biométrique ou sensible n'est collectée.

Pas de cookies, pas de tracking, pas d'analytics

L'application n'utilise aucun cookie de suivi, aucun pixel publicitaire, aucun outil d'analyse comportementale (Google Analytics, Matomo, etc.). Les identifiants professeur sont mémorisés uniquement dans le localStorage local au navigateur, jamais envoyés à un tiers.

Transit via relais souverain (n8n Éducation Nationale)

Les requêtes transitent par un proxy n8n hébergé sur n8n.incubateur.education.gouv.fr — un service de l'Incubateur de l'Éducation Nationale. Aucun service tiers étranger n'est impliqué. L'IA Albert est également souveraine (Étalab / État français).

Droit à l'effacement garanti

La suppression d'une session depuis l'interface professeur efface le fichier session et tous les résultats élèves associés du Nuage. Aucune donnée résiduelle. Vous pouvez également supprimer l'ensemble du dossier experia/ depuis votre Nuage pour un effacement complet.

Votre responsabilité en tant qu'enseignant

En utilisant Experia, vous êtes responsable de traitement des données de vos élèves stockées sur votre Nuage. Informez vos élèves que leurs résultats sont sauvegardés (obligation d'information RGPD — art. 13). Vous devez conserver ces données uniquement le temps nécessaire à l'évaluation, puis supprimer la session.
