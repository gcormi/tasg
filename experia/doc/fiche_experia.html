Circuit des données — qui fait quoi, où va l'information
① Création de la séance — côté professeur
👨‍🏫 Professeur → saisit un thème, un niveau ou colle une leçon dans l'interface Experia
↓ ⚙️ Worker Cloudflare → 🤖 Albert API (Étalab) → génère les activités : QCM, mots croisés, phrases à trous, mots mêlés…
↓ 💾 Cloudflare KV → la session est stockée + un lien unique est généré pour les élèves
② Activité élève — en classe ou à distance
🎓 Élève → ouvre le lien dans son navigateur (tablette, téléphone, PC) — sans compte, sans installation
↓ choisit un pseudonyme libre → accède aux activités → répond
↓ ⚙️ Worker Cloudflare → 🤖 Albert API → corrige la réponse et génère un feedback personnalisé
↓ 🎓 Élève ← reçoit le feedback immédiatement dans son navigateur — résultats enregistrés dans KV
③ Suivi en temps réel — côté professeur
💾 Cloudflare KV → ⚙️ Worker → 👨‍🏫 Professeur ← voit les résultats en direct, peut réactiver une activité, télécharger la correction (.doc)
Infrastructures mobilisées
IA — génération & correction
Albert API
IA souveraine française (Étalab / DINUM). Les échanges restent sur les serveurs de l'État français.
Proxy & stockage des sessions
Cloudflare Workers + KV
Accord DPA RGPD en place. Données localisables en Europe. Effacement possible à tout moment.
Hébergement de l'interface
Forge des Communs
forge.aeif.fr — GitLab de l'Éducation Nationale. Déploiement automatique, code open source.
