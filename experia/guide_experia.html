Présentation

Qu'est-ce qu'Experia ?

Experia est un outil pédagogique qui permet aux professeurs de créer des séances d'évaluation enrichies par l'intelligence artificielle. Les élèves répondent à des activités (questions ouvertes, QCM, textes à trous…), l'IA corrige et fournit un retour immédiat.

📝

Pour les professeurs

Créez des sessions en quelques minutes, partagez un lien avec vos élèves, consultez les résultats en temps réel.

🎓

Pour les élèves

Accès immédiat via un simple lien — aucune inscription, aucun compte, aucune application à installer.

🤖

L'IA Albert

Correction automatique et feedback personnalisé grâce à Albert, l'IA souveraine française hébergée par l'État.

🔒

Données protégées

Aucun mot de passe Nuage ni clé API stocké côté serveur. Conforme RGPD dès la conception.

Pour commencer, vous avez besoin de :
• L'URL d'accès fournie par votre référent Experia (contactez-le directement)
• Le code d'inscription fourni par votre référent Experia
• Votre clé Albert (créée sur albert.api.etalab.gouv.fr)
Accès par URL : Experia fonctionne entièrement dans le navigateur — aucun logiciel à installer. Votre référent vous communique les liens d'accès. Vous pouvez les mettre en favori.

Guide Professeur — Étape 1

Créer son compte

La création de compte se fait en une seule fois. Votre référent Experia vous communique le lien d'accès et le code d'inscription — aucun fichier à télécharger, aucune installation.

1

Ouvrez le lien de création de compte

Votre référent Experia vous envoie le lien par e-mail ou messagerie. Ouvrez-le dans votre navigateur habituel.

2

Entrez le code d'inscription

Ce code vous a été communiqué par votre référent Experia. Sans ce code, la création de compte est impossible (sécurité).

3

Choisissez votre identifiant

Exemple : dupont.marie ou mmartinez. Lettres, chiffres, tirets et points uniquement. Notez-le précieusement.

4

Choisissez votre secret (mot de passe)

Minimum 6 caractères. Ce secret vous sera demandé à chaque connexion. Notez-le — il ne peut pas être récupéré.

5

Cliquez "Créer mon compte"

En cas de succès, vous verrez votre identifiant et l'URL du Worker. Ces informations sont automatiquement mémorisées pour la prochaine connexion.

Important : notez votre identifiant et votre secret dans un endroit sûr. En cas de perte du secret, vous devrez contacter votre référent Experia pour réinitialiser votre compte.
Guide Professeur — Étape 2

Se connecter

Ouvrez le lien Experia Professeur fourni par votre référent dans votre navigateur. Si vous avez déjà créé votre compte sur le même appareil, votre identifiant est pré-rempli. Mettez ce lien en favori — c'est votre accès permanent.

1

Ouvrez le lien Experia Professeur

Le lien vous a été transmis par votre référent Experia. Ajoutez-le à vos favoris pour y accéder rapidement à chaque séance.

2

Vérifiez l'identifiant pré-rempli

Votre identifiant devrait être déjà renseigné. Sinon, entrez-le manuellement.

3

Entrez votre secret et cliquez "Se connecter"

Le secret n'est jamais mémorisé — vous devez le retaper à chaque connexion. C'est une mesure de sécurité intentionnelle.

Connexion rapide : vous pouvez générer un "lien de connexion rapide" depuis prof.html. Ce lien contient votre identifiant pré-rempli et peut être mis en favori. Attention : il ne contient pas votre secret, vous devrez toujours le saisir.

Changer de poste ou d'appareil

Si vous travaillez depuis un nouvel appareil, rouvrez simplement le lien Experia Professeur (favori ou mail de votre référent) et entrez votre identifiant + secret. L'URL du serveur est déjà intégrée au lien.

Guide Professeur — Étape 3

Créer une session

Une session est un ensemble d'activités que vous proposez à vos élèves pour une séance donnée. Chaque session génère un code unique et un lien élève.

1

Cliquez "+ Nouvelle session"

Depuis le tableau de bord, cliquez sur le bouton bleu en haut à droite.

2

Donnez un titre à votre session

Exemple : "Évaluation chapitre 3 — La Révolution française". Ce titre est visible sur le tableau de bord.

3

Choisissez les activités

Sélectionnez le type d'activités (QCM, questions ouvertes, texte à trous…) et renseignez leur contenu. L'IA utilisera ces données pour corriger les élèves.

4

Configurez le modèle IA (optionnel)

Par défaut, le modèle partagé est utilisé. Si vous avez votre propre clé Albert, vous pouvez choisir un modèle plus puissant.

5

Validez — la session est créée

Le lien élève est généré automatiquement. La session est sauvegardée dans le serveur central et accessible depuis n'importe quel appareil.

Guide Professeur — Étape 4

Inviter les élèves

Une fois la session créée, un lien élève unique est généré. Ce lien contient tout ce dont l'élève a besoin — aucune inscription, aucun compte.

1

Copiez le lien élève

Dans la vue session, cliquez sur le bouton "Copier le lien". Vous pouvez aussi afficher le QR Code pour une projection en classe.

2

Partagez le lien

Envoyez-le par e-mail, ENT, Pronote, ou projetez le QR Code. L'élève n'a qu'à cliquer — aucune installation requise.

3

L'élève entre son prénom/pseudonyme

À l'ouverture du lien, l'élève choisit comment il souhaite être identifié dans les résultats.

Sécurité : le lien élève ne contient aucune donnée sensible. Il contient uniquement l'URL du serveur et l'identifiant de la session. Vos identifiants professeurs ne sont jamais inclus.

Ce que voit l'élève

L'élève accède à une interface claire adaptée au mobile et à l'ordinateur. Il répond aux activités, reçoit un feedback immédiat de l'IA, et peut voir son score global à la fin.

Guide Professeur — Étape 5

Consulter les résultats

Les résultats des élèves sont visibles en temps réel dans prof.html, dans la vue de la session concernée.

1

Ouvrez la session dans le tableau de bord

Cliquez sur la session dans la liste. Les résultats se chargent automatiquement.

2

Consultez les résultats par élève

Chaque élève apparaît avec son score global et le détail de ses réponses. Cliquez sur un élève pour voir le détail.

3

Ajoutez une note manuelle (optionnel)

Sur la fiche d'un élève, vous pouvez ajouter votre propre note /20 et un commentaire, en complément de la correction IA.

4

Téléchargez la correction (optionnel)

Utilisez le bouton "Télécharger la correction" pour exporter un récapitulatif au format HTML.

Notice Technique

Déploiement Cloudflare Worker

Le Worker Experia est le composant central. Il est hébergé sur Cloudflare Workers (gratuit jusqu'à 100 000 requêtes/jour). Voici les étapes de déploiement initial.

Prérequis

  • Un compte Cloudflare gratuit (dash.cloudflare.com)
  • Le fichier experia_worker_v2.js
  • Une clé Albert API (albert.api.etalab.gouv.fr)

Étape 1 — Créer le Worker

1

Aller dans Compute → Workers & Pages → Create

Choisissez "Create Worker", donnez-lui le nom experia-proxy, puis cliquez "Deploy".

2

Coller le code du Worker

Cliquez "Edit code", sélectionnez tout (Ctrl+A), collez le contenu de experia_worker_v2.js, puis "Deploy".

Étape 2 — Créer le KV namespace

1

Storage & databases → KV → Create namespace

Nommez-le exactement EXPERIA_KV. Cliquez "Add".

2

Lier le KV au Worker

Workers & Pages → experia-proxy → Bindings → Add binding → KV namespace.
Variable name : EXPERIA_KV — KV namespace : EXPERIA_KV. Puis "Save".

Étape 3 — Ajouter les secrets

Dans Workers & Pages → experia-proxy → Settings → Variables and Secrets → Add :

VariableTypeValeur
ALBERT_KEY Secret Votre clé Albert API (JWT commençant par eyJ…)
MASTER_SECRET Secret Code d'inscription que vous donnerez aux profs (ex: experia2026)

Cliquez "Deploy" après chaque ajout.

Notice Technique

Secrets & Variables

Le Worker utilise deux types de secrets :

SecretRôleQui le connaît
ALBERT_KEY Clé API Albert partagée pour les appels IA des élèves Référent Experia uniquement (stocké en secret Cloudflare)
MASTER_SECRET Code d'inscription — gate d'accès à la création de comptes profs Référent Experia + profs du groupe test
Rotation du MASTER_SECRET : si le code d'inscription est compromis, changez-le dans les secrets Cloudflare. Les comptes déjà créés ne sont pas affectés (ils sont stockés dans KV avec leur hash individuel).

Authentification professeur

Les professeurs s'authentifient avec profId + profSecret. Le profSecret n'est jamais stocké en clair — seul son hash SHA-256 est conservé dans KV. Même le référent Experia ne peut pas lire les secrets des profs.

Notice Technique

Maintenance

Mettre à jour le Worker

1

Cloudflare → Workers & Pages → experia-proxy → Edit code

Sélectionnez tout (Ctrl+A), collez le nouveau code, cliquez "Deploy". Les sessions existantes ne sont pas affectées.

Réinitialiser le secret d'un prof

1

Storage & databases → KV → EXPERIA_KV

Cherchez la clé prof:{profId} et supprimez-la. Le prof pourra ensuite recréer son compte via setup_experia.html avec le même identifiant.

Supprimer les données d'un prof

Dans KV, supprimez :

  • prof:{profId} — le compte
  • manifest:{profId} — la liste des sessions
  • Chaque session:{sessionId} listé dans le manifeste
  • Chaque resultlist:{sessionId} et result:{sessionId}:*

Notice Technique

Conformité RGPD

Experia a été conçu avec la protection des données comme contrainte principale, pas comme option.

🔑

Secrets profs jamais stockés

Les mots de passe Nuage et clés Albert des profs ne transitent que dans le navigateur du prof. Le serveur ne les voit jamais.

🔒

Hash uniquement

Le secret prof est hashé en SHA-256 avant stockage. Irréversible — même le référent Experia ne peut pas le lire.

👤

Élèves anonymes

Les élèves ne créent pas de compte. Ils choisissent librement leur pseudonyme — aucun e-mail, aucune identité réelle requise.

🇫🇷

IA souveraine

Les réponses des élèves sont envoyées à Albert API (hébergée par l'État français), pas à des services US ou hors UE.

Ce qui est stocké côté serveur

DonnéeStockée ?Forme
Mot de passe Nuage du prof❌ Non—
Clé Albert du prof❌ Non—
Secret prof✅ OuiHash SHA-256 uniquement
Sessions (questions, barème)✅ OuiJSON dans KV

Conclusion : pour une phase test de 20 à 30 professeurs, Experia est conforme aux principes RGPD de minimisation des données, de pseudonymisation et de souveraineté des traitements IA. Pour un déploiement institutionnel, un hébergement sur serveur DSI est recommandé.
