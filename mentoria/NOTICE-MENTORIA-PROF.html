Le Guide Ultime de l'Enseignant

Bienvenue dans Mentoria — studio de création d'assistants pédagogiques IA souverains.

✍️ Conçu et développé par Gilles Cormi, enseignant (académie de Corse) — gcormi.forge.apps.education.fr/frojet/mentoria  ·  v2.7 — Avril 2026

1. Philosophie & Promesse de Mentoria

Mentoria est né d'une vision : offrir aux enseignants "le meilleur des deux mondes". L'outil combine la structure guidée d'un cours scénarisé (via des boutons d'actions) et la fluidité de l'intelligence artificielle pour un dialogue naturel avec l'élève.

🛡️ 100% Souverain et RGPD by design

Mentoria se distingue de toutes les autres plateformes du marché :

  • Aucune base de données tierce : Il n'y a ni serveurs Google, ni serveurs Firebase. Vos assistants sont sauvegardés sous forme de fichiers dans votre propre espace institutionnel (Nuage ou Forge).
  • Sans compte élève : Les élèves n'ont rien à installer et ne créent pas de compte. Ils s'identifient uniquement avec leur identifiant ENT de manière déclarative.
  • IA de l'État : Le "cerveau" utilisé est Albert, développé par Etalab (la direction interministérielle du numérique).

2. Prérequis : Ce dont vous avez besoin

Pour faire fonctionner Mentoria, vous devez vous munir de deux "clés" essentielles avant votre première utilisation.

A. Une clé API Albert (Le cerveau)

C'est ce qui autorise votre assistant à interroger l'intelligence artificielle du gouvernement.

  1. Rendez-vous sur le site officiel : albert.etalab.gouv.fr (Playground Albert).
  2. Connectez-vous via ProConnect avec vos identifiants académiques.
  3. Dans le menu, cherchez la section API Keys et cliquez sur "Générer une nouvelle clé".
  4. Attention : Copiez cette clé immédiatement (elle commence par sk-...). Elle ne s'affichera qu'une seule fois ! Stockez-la dans un gestionnaire de mots de passe.

B. Un accès "Nuage" (Le stockage)

C'est là que vos créations seront sauvegardées. Pour des raisons de sécurité, vous ne devez pas utiliser votre mot de passe habituel, mais créer un "mot de passe d'application".

  1. Connectez-vous sur nuage.apps.education.fr.
  2. Regardez la barre d'adresse de votre navigateur et notez le nom exact de votre serveur (ex : nuage03.apps.education.fr).
  3. Cliquez sur votre nom/avatar en haut à droite, puis sur Paramètres.
  4. Dans le menu de gauche, cliquez sur Sécurité.
  5. Allez tout en bas de la page jusqu'à "Appareils et sessions". Dans le champ "Nom de l'application", tapez Mentoria puis cliquez sur Créer un nouveau mot de passe d'application.
  6. Copiez le mot de passe généré (une suite de lettres générée aléatoirement).

3. Les 3 modes de Stockage

Dès l'ouverture du Studio Mentoria, un écran d'accueil vous demande où vous souhaitez travailler. L'outil s'adapte à vos contraintes réseaux et à votre profil.

Mode Idéal pour... Fonctionnement & Contraintes
☁️ Nuage apps.edu
(Recommandé)		 La grande majorité des enseignants. Vos bots sont sauvés sur votre espace Nuage institutionnel (via WebDAV). Accessible depuis le collège ou la maison. Nécessite le mot de passe d'application et l'URL d'instance (voir section 2).
💾 Clé USB / Local Les zones sans internet ou les salles mal équipées. Le Studio lit et écrit les fichiers directement dans un dossier de votre clé USB ou disque dur (via File System Access API). Attention : Fonctionne uniquement sur Google Chrome ou Microsoft Edge (pas Firefox).
🦊 La Forge Les profils techniques / formateurs numériques. Se connecte à votre dépôt GitLab EN. Nécessite un "Badge" (Jeton d'accès) et l'ID de votre projet. Utile pour versionner ses bots.

4. Le Studio : Créer un assistant pas à pas

L'interface de création (le Studio) est divisée en 4 blocs principaux. La sauvegarde se fait manuellement via le bouton Enregistrer en haut à droite.

Étape 1 : Identité & Présentation

C'est l'enrobage visuel de votre assistant.

  • L'Avatar : Ajoutez une image. Mentoria la redimensionnera automatiquement (max 128x128) pour qu'elle reste légère (essentiel pour le lien de partage).
  • Titre du projet : Le nom "administratif" du bot (ex: "Maths 3e Chap 4").
  • Nom de l'assistant : Ce que l'élève verra (ex: "MathBot").
  • Message de bienvenue : La toute première bulle de dialogue affichée à l'ouverture. Astuce : Vous pouvez utiliser la variable {name} pour que le bot dise "Bonjour j.dupont !".

Étape 2 : Pédagogie & Règles

C'est le "cerveau" comportemental de votre IA.

  • Instructions : Le prompt système principal (ex: "Tu es un professeur de SVT s'adressant à des 6e...").
    ⚠️ Limite conseillée : 200 mots maximum. Au-delà, l'IA risque de "diluer" son attention et d'ignorer certaines consignes.
  • Source des réponses :
    • Corpus seul : L'IA refusera catégoriquement de répondre si l'information n'est pas dans vos documents. Très strict, idéal pour des QCM de lecture.
    • Les deux (Recommandé) : L'IA privilégie vos documents mais s'autorise à utiliser sa culture générale pour expliquer un concept.
    • Albert seul : L'IA ignore vos documents.
  • Règles & Interdictions : Mentoria propose 20 règles pré-configurées (ex: "Ton encourageant", "Ne jamais donner la réponse", "Refuser le hors-sujet"). Cochez celles qui correspondent à votre posture pédagogique (Posture de Maïeutique).
  • Votre propre règle : Vous pouvez inventer une règle libre (ex: "Termine toujours tes phrases par un emoji" ou "Vouvoie l'élève").

Étape 3 : Corpus de Connaissance

Ce sont les documents que l'IA va "lire" pour construire ses réponses.

⚠️ Limites techniques du Corpus

Poids maximal : Ne dépassez pas 200 Ko de texte pur (environ 40 à 50 pages). Si le corpus est trop volumineux, l'URL de publication sera trop lourde et la page plantera côté élève.

Privilégiez la précision : Ne donnez pas le manuel entier. Donnez uniquement la fiche de synthèse du chapitre en cours.

  • Fichiers PDF / TXT : Vous pouvez uploader jusqu'à 5 fichiers. Mentoria extraira le texte automatiquement.
  • URLs de référence : Vous pouvez coller l'URL d'une page Web (ex: une page Wikipédia). Mentoria ira lire et importer le texte de cette page. Note : Cela ne fonctionne que pour des pages HTML classiques, pas pour des liens menant vers un fichier PDF.

Étape 4 : Boutons Rapides & API

Les Boutons Rapides (ou Quick Replies) sont une exclusivité Mentoria. Ils permettent de guider l'élève sans qu'il ait besoin de taper au clavier.

  • Fonctionnement "One-Shot" : Lorsqu'un élève clique sur un bouton (ex: "Donne-moi un indice"), Mentoria envoie une consigne cachée très stricte à l'IA pour qu'elle s'exécute immédiatement, puis qu'elle reprenne son comportement normal ensuite.
  • Vous pouvez piocher dans la bibliothèque intégrée (Aide, Approfondissement, Évaluation, Navigation) ou créer vos propres boutons personnalisés.

Clé API : C'est ici, tout en bas, que vous collez la clé `sk-...` obtenue à la section 2. Elle est mémorisée localement sur votre ordinateur pour vos prochaines sessions.

5. Publier, Partager et Modifier

Comment fonctionne la publication ?

Mentoria utilise une technologie d'URL auto-portée (encodage Base64). Lorsque vous cliquez sur "Publier", le logiciel compresse l'intégralité de votre configuration (titre, règles, textes du corpus, image de l'avatar, clé API chiffrée) et l'intègre directement à l'intérieur du lien Web.

Pourquoi ce choix technique ?

Cela garantit une souveraineté absolue. Aucun serveur commercial n'héberge la configuration de votre bot. L'URL est le bot. C'est pourquoi le lien généré est très long.

Déployer en classe

  1. Cliquez sur Publier.
  2. Copiez le long lien généré (https://.../compagnion.html#config=...).
  3. Notez le Code de Groupe (ex: X7M9) qui s'affiche à l'écran. Il est indispensable pour suivre cette classe spécifique.
  4. Distribuez le lien aux élèves via le cahier de texte de l'ENT, un document partagé, ou via un générateur de QR Code.

⚠️ Règle d'Or de la Modification

Puisque toute l'intelligence du bot est contenue dans le lien, si vous modifiez votre bot dans le Studio, l'ancien lien ne se mettra pas à jour !

Vous devez systématiquement cliquer à nouveau sur "Publier", ce qui générera un nouveau lien et un nouveau code de groupe à distribuer aux élèves.

6. Le Suivi Pédagogique (Console Professeur)

En haut à droite du Studio, le bouton Suivi ouvre la console de supervision. Cette fonctionnalité utilise un "relais souverain" (n8n) hébergé par l'État pour faire transiter temporairement les messages.

  1. Ouvrez la console de Suivi.
  2. Sélectionnez le Code de groupe correspondant à la séance que vous venez de lancer.

Ce que vous voyez à l'écran :

L'interface de suivi est divisée en deux parties pour vous permettre une supervision granulaire et précise des échanges :

  • La liste de la classe (panneau de gauche) : Elle s'actualise automatiquement toutes les 30 secondes. Pour chaque élève connecté, vous visualisez d'un coup d'œil son identifiant ENT, le volume d'échanges (nombre de messages), l'heure de sa dernière activité, ainsi qu'un aperçu tronqué de sa dernière interaction.
  • Le fil de conversation (panneau central) : En cliquant sur le nom d'un élève dans la liste, vous déroulez l'intégralité de son dialogue avec l'assistant. Vous lisez exactement le texte tapé par l'élève (dans les bulles de couleur) et la réponse formulée par l'IA (dans les bulles blanches), avec l'horodatage précis de chaque intervention.

💡 Intérêt Pédagogique de la lecture des échanges

Lire les conversations en direct ou en fin de séance vous offre de précieux indicateurs :

  • Repérer les blocages : Un élève qui tourne en rond ou n'arrive pas à utiliser le vocabulaire attendu pour faire avancer l'IA.
  • Évaluer le comportement de votre IA : Vérifier si le bot est trop permissif ou s'il applique bien la consigne de maïeutique (faire réfléchir sans donner la solution).
  • Cadrer la séance : Identifier très vite les élèves qui tentent de contourner les règles, de piéger le bot, ou de faire du hors-sujet.

Actions de modération :

  • Bouton "Expulser" (🚫) : Coupe immédiatement l'accès de l'élève à l'assistant. Son interface se bloquera.
  • Bouton "Supprimer tous les logs" (🗑️) : Efface définitivement l'historique des conversations du groupe.

Conformité RGPD

Les conversations sont conservées temporairement pour des raisons pédagogiques (maximum 7 jours) sur l'infrastructure de l'État. Il est de la responsabilité de l'enseignant de cliquer sur la "Corbeille" en fin de séance s'il n'a plus besoin des traces.

7. L'Interface Élève (Le Compagnon)

L'interface de l'élève a été pensée pour minimiser la charge cognitive. Aucune installation n'est requise (fonctionne sur PC, Tablette ou Smartphone dans le navigateur web).

  1. L'écran de connexion : L'élève saisit obligatoirement son Identifiant ENT (ex: j.dupont). Une mention légale l'avertit que le professeur peut lire les échanges.
  2. L'espace de Chat : Si l'élève se reconnecte plus tard (avec le même identifiant et le même lien), l'IA se souviendra de lui et restaurera la conversation précédente.
  3. Les boutons d'action : Ils apparaissent juste au-dessus de la barre de texte, permettant à l'élève de rebondir sans effort de frappe.
  4. Export : L'élève dispose d'un bouton (icône de téléchargement 📥) en haut à droite pour télécharger l'intégralité de sa discussion au format .txt et la garder dans ses notes personnelles.

8. Dépannage & FAQ Technique

Problème : "Erreur 405 lors de la connexion au Nuage"

Cause : Le Studio Mentoria essaie de se connecter au serveur Nuage général, mais votre compte académique est assigné à un serveur spécifique.

Solution : Connectez-vous "manuellement" sur votre Nuage via un nouvel onglet. Regardez la barre d'adresse. Vous y lirez quelque chose comme nuage02.apps.education.fr. C'est cette adresse exacte qu'il faut saisir dans le champ "Instance Nuage" du Studio.

Problème : Les élèves ont un écran rouge "Lien Invalide"

Cause : Le lien a été tronqué (coupé) lors du copier-coller dans l'ENT ou le logiciel de messagerie.

Solution : Assurez-vous de sélectionner l'intégralité du lien (utilisez le bouton "Copier" du Studio). Si le problème persiste (l'ENT limite la taille des liens), utilisez un réducteur d'URL (comme Frama.link ou bzh.link) ou transformez le lien en QR Code.

Problème : L'IA ne répond pas, tourne en boucle, ou affiche "Erreur API"

Cause : Soit la clé API est expirée, soit le corpus est trop lourd et sature le contexte d'Albert.

Solution : Allez dans la Section 4 du Studio, supprimez l'ancienne clé et collez une nouvelle clé fraîchement générée sur le Playground. Assurez-vous également d'avoir vidé les fichiers PDF inutiles (gardez l'essentiel).

Problème : Je suis sur Clé USB et je dois tout le temps re-sélectionner mon dossier

Cause : C'est une sécurité normale imposée par les navigateurs web (Chrome/Edge). Un site web ne peut pas mémoriser un accès permanent à un dossier physique local.

Solution : Il n'y en a pas, c'est le comportement attendu. L'avantage est que Mentoria ne laisse aucune trace de vos bots sur un PC partagé (en salle des profs par exemple).

9. Bonnes pratiques pédagogiques

  • Corpus ciblé : Donnez le cours du jour, pas le manuel entier. Un corpus de 2-3 pages ciblées est bien plus efficace qu'un document de 80 pages.
  • Format TXT de préférence : Exportez vos cours en TXT plutôt qu'en PDF — plus léger et mieux lu par Albert.
  • Tester avant diffusion : Ouvrez le lien élève vous-même et posez quelques questions avant de le distribuer en classe.
  • Un assistant par séquence : Créez un assistant différent pour chaque chapitre ou séance plutôt qu'un bot généraliste.
  • Activer "Hors sujet → rediriger" : Indispensable pour cadrer les échanges et éviter les dérives.
  • Maïeutique : Activez "Ne jamais donner la réponse directement" pour forcer l'élève à réfléchir — c'est la posture pédagogique la plus efficace.
  • Instructions courtes : Restez sous 200 mots dans le champ Instructions. Au-delà, Albert commence à ignorer une partie des consignes.
  • Conseil RGPD : Si vous utilisez le mode "pseudonyme" (sans suivi de classe), demandez aux élèves d'utiliser un identifiant neutre — évitez nom + prénom + classe combinés.
