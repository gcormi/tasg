Feuille de Route & Vision

Prochaines étapes, ambitions et philosophie de Mentoria.

✍️ Conçu et développé par Gilles Cormi, enseignant (académie de Corse) — gcormi.forge.apps.education.fr/frojet/mentoria  ·  v2.7 — Avril 2026

🚧 Work In Progress — Outil en évolution continue

Mentoria est un projet vivant développé en méthode Agile. Les éléments présentés dans cette feuille de route représentent le cap stratégique actuel. L'outil est constamment affiné grâce aux retours des utilisateurs sur le terrain et à la coordination technique avec les services de l'État (Incubateur EN et DINUM).

1. La Vision : "Le meilleur des deux mondes"

L'objectif fondamental de Mentoria est de créer un hybride parfait pour l'enseignement, combinant la rigueur structurelle d'un cours scénarisé et la fluidité d'un dialogue naturel avec l'intelligence artificielle.

  • La structure : Permettre à l'enseignant de garder le contrôle. Mentoria intègre des boutons de choix rapides pour guider l'élève, et prévoit à terme d'intégrer des "Missions" ou des étapes fixes dans la progression pédagogique.
  • L'Intelligence (L'héritage Albert) : Offrir une fluidité conversationnelle. L'IA Albert gère le dialogue libre entre les étapes imposées, en appliquant des règles strictes de maïeutique pour faire réfléchir l'élève sans jamais "faire à sa place". L'ancrage sur le corpus de l'enseignant permet de réduire drastiquement les hallucinations.

2. Philosophie Institutionnelle & Conformité RGPD

Mentoria n'est pas qu'un outil technique, c'est une démarche politique et pédagogique forte. Il a été pensé avec l'aide des correspondants ministériels pour s'inscrire parfaitement dans le cadre de confiance de l'institution.

Les piliers du projet :

  • Souveraineté Absolue : Utilisation exclusive des infrastructures de l'État. Exit les géants du web (Google, Firebase, AWS). Le tryptique fondateur est : Nuage EN, L'IA Albert (Etalab), et La Forge EN.
  • RGPD by design : Le principe de minimisation des données est poussé à l'extrême. Les élèves n'ont pas de compte, et il n'y a aucune base de données centrale qui piste leurs habitudes.
  • Universalité et Accessibilité : L'outil a vocation à être utilisable par tout enseignant, même sans bagage technique, en s'appuyant uniquement sur ses identifiants académiques existants.

3. Jalons Franchis (Ce qui a été réalisé)

Le passage à la v2 souveraine a permis de concrétiser les fondations techniques de Mentoria :

  • ✓ Suppression totale des dépendances commerciales : Retrait complet de Firebase. L'application fonctionne de manière autonome en s'appuyant sur des fichiers JSON (Base de données orientée documents locaux).
  • ✓ Moteurs de stockage souverains : Intégration de l'API WebDAV pour la synchronisation sur les espaces Nextcloud de l'Éducation Nationale, et de l'API GitLab pour La Forge.
  • ✓ Mode 100% hors-ligne (Air-gap) : Support de l'API File System Access permettant à un professeur en zone blanche de sauvegarder ses bots sur clé USB.
  • ✓ Publication décentralisée (LZ-String Base64) : Création d'une URL auto-portée. La configuration de l'assistant (règles, corpus, avatar) est encapsulée directement dans le lien élève via LZ-String + Base64, supprimant le besoin d'un serveur d'hébergement.
  • ✓ Suivi de classe en temps réel : Mise en place du système de supervision des logs (effaçables) via le relais proxy souverain n8n.
  • ✓ Bouton "Changer de stockage" : Retour à l'écran d'accueil sans rechargement de page ni perte des identifiants mémorisés.
  • ✓ Indicateur de synchronisation : Badge animé visible lors de toute lecture ou écriture vers le Nuage / la Forge.
  • ✓ Aide intégrée (icônes ?) : Tooltips contextuels au survol sur chaque champ complexe du Studio.
  • ✓ Jauge de poids du bot : Barre en temps réel dans l'éditeur et dans le modal de publication — verte / orange / rouge selon le poids réel du lien encodé.
  • ✓ Noms de fichiers lisibles (slugify) : Les bots JSON sont nommés d'après leur titre (ex: bot_pink_floyd.json) au lieu d'un timestamp.
  • ✓ Résilience aux fichiers manquants (Nuage) : Si un fichier .json est supprimé directement depuis le Nuage, la bibliothèque se charge quand même et signale les absents.
  • ✓ Export / Import unitaire : Téléchargement d'un bot en .json pour le partager avec un collègue ; import du fichier JSON dans la bibliothèque.
  • ✓ Niveaux de difficulté : Sélecteur 🌱 Débutant / 📚 Standard / 🎯 Avancé injecté dans le prompt système du Compagnon.
  • ✓ Templates de posture : 5 profils pré-configurés (Le Socratique, Le Coach, L'Animateur de débat, Le Tuteur de révision, Le Challenger).
  • ✓ Grille d'évaluation visible côté élève : Sur l'écran d'accueil du Compagnon, l'élève voit les 5 critères d'évaluation et leur barème (/10) avant de commencer sa conversation — section repliable.
  • ✓ Évaluation IA de la conversation (côté prof) : Dans la console de suivi, un bouton "Évaluer" analyse la conversation d'un élève et génère une note indicative sur 10 points répartis sur 5 critères, avec commentaire par critère et appréciation générale.

4. Objectifs à court terme (Phase de test)

Les prochaines semaines sont dédiées à la validation en conditions réelles et aux 2 dernières fonctionnalités pédagogiques avancées.

Priorité 1 : Tests et Validation en conditions réelles

  • Tests de charge Nuage : Vérification de la résilience du proxy n8n face à une classe de 35 élèves écrivant simultanément leurs logs sur le serveur Nextcloud de l'enseignant.
  • Compatibilité Navigateurs : Validation du décodeur compagnion.html sur d'anciens navigateurs ou tablettes souvent présents en établissement.
  • Gestion des avatars : Tests d'encodage avec différents formats d'images (PNG, JPG, AVIF, WebP) pour garantir la légèreté des URLs.
  • □ Robustesse des erreurs : Messages d'erreur très clairs si le Nuage est mal configuré ou si l'API Albert est saturée (retry automatique).
  • □ Externalisation avatar/clé API : Stocker ces éléments lourds séparément dans un fichier maître sur le Nuage — le lien élève ne contiendrait plus qu'un identifiant léger.
  • □ Exit Quiz : Questionnaire de fin de session dans le Compagnon, avec score intégré dans l'export .txt élève.

5. Évolutions Techniques & Pédagogiques Futures

La vision à moyen/long terme pour Mentoria vise à étendre ses capacités sans sacrifier sa légèreté.

A. Ergonomie et Interface — réalisé

  • ✓ Bouton "Changer de stockage" : Retour à l'écran d'accueil sans rechargement ni perte des identifiants mémorisés.
  • ✓ Indicateur de synchronisation : Badge animé visible lors de toute opération de lecture/écriture.
  • ✓ Jauge de poids URL : Jauge en temps réel dans le Studio et dans le modal de publication — vert / orange / rouge.
  • ✓ Aide intégrée (icônes ?) : Tooltips contextuels sur chaque champ complexe du Studio.

B. Collaboratif & Échanges — réalisé

  • ✓ Export / Import unitaire : Téléchargement d'un bot en .json depuis la sidebar pour le partager avec un collègue ; import depuis un fichier .json reçu.

C. Pédagogie avancée — réalisé

  • ✓ Niveaux de difficulté : Sélecteur Débutant / Standard / Avancé dans le Studio, injecté dans le prompt système du Compagnon.
  • ✓ Templates de posture : 5 profils pédagogiques pré-configurés (Le Socratique, Le Coach, L'Animateur de débat, Le Tuteur de révision, Le Challenger).

D. Optimisation réseau — en cours

  • □ Externalisation avatar/clé API : Stocker l'avatar et la clé dans un fichier maître sur le Nuage. Le lien élève ne contiendrait plus qu'un identifiant léger, résolvant définitivement les problèmes de lien trop long.

E. Pédagogie avancée — à venir

  • □ Exit Quiz : Questionnaire de vérification en fin de session dans le Compagnon ; score inclus dans l'export .txt élève.

F. Aide intégrée — à venir

  • □ Tutoriel visuel au démarrage : Guide en 3 étapes sur l'écran de connexion pour les enseignants novices (clé API, mot de passe d'application Nuage, premier bot).
  • □ Badge de statut connectivité : Indicateur Vert / Rouge pour signaler en temps réel si le relais n8n et le Nuage sont opérationnels.

G. Évaluation & Suivi élève — à venir

  • □ Catalogue de templates de bots : 5 à 6 modèles prêts à l'emploi (bot de révision, bot de débat, bot correcteur d'orthographe…) accessibles directement depuis le Studio pour démarrer sans configuration.
  • □ Retour au l'élève en fin de session : À la fermeture du Compagnon, l'élève reçoit un résumé court généré par le bot — "Voici ce que le bot a remarqué dans ta démarche…" — avec un commentaire encourageant sur ses points forts et une piste de progrès.
  • □ Export rapport de classe : Vue synthétique téléchargeable (PDF ou tableur) récapitulant la participation, la distribution des notes indicatives et les élèves n'ayant pas encore interagi — utile pour la trace pédagogique.
  • □ Mode test avant publication : Bouton "Tester ce bot" dans le Studio permettant au professeur de dialoguer lui-même avec son bot avant de partager le lien aux élèves — pour vérifier le rôle, les limites et le ton du personnage.
