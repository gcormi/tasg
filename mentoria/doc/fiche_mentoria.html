🇫🇷 Albert API — IA souveraine ☁️ Nuage national (apps.education.fr) 🔧 n8n incubateur.education.gouv.fr 🏫 Forge des Communs Numériques
Circuit des données — qui fait quoi, où va l'information
① Création du bot — côté professeur
👨‍🏫 Professeur → remplit un formulaire guidé : rôle du bot, personnalité, limites, contexte pédagogique
↓ ☁️ Nuage national ← la configuration du bot est stockée sur le Nuage de l'Éducation Nationale (WebDAV)
↓ 🏫 Forge (interface) → un lien de classe unique est généré — le professeur le partage avec ses élèves
② Dialogue élève avec le bot — en classe ou à distance
🎓 Élève → ouvre le lien — voit la grille d'évaluation (5 critères / 10 pts) sur l'écran d'accueil avant de commencer
↓ saisit un message dans le chat → envoyé à l'orchestrateur
↓ ⚙️ n8n (incubateur EN) → récupère la config du bot sur le Nuage + transmet le contexte à Albert
↓ 🤖 Albert API (Étalab) → génère la réponse du bot en respectant son rôle et ses limites pédagogiques
↓ ⚙️ n8n → ☁️ Nuage national → la conversation est sauvegardée sur le Nuage EN
↓ 🎓 Élève ← reçoit la réponse du bot dans son navigateur
③ Suivi en temps réel — côté professeur
☁️ Nuage national → 🏫 Interface suivi → 👨‍🏫 Professeur ← voit toutes les conversations en direct, peut intervenir, exclure un élève, ou lancer une évaluation IA (note /10 sur 5 critères, indicative)

Positionnement — entre deux outils existants
ChatMD
Bot configurable via fichier Markdown. Très simple mais peu souple, pas de suivi en temps réel.
✦ Mentoria
Entre les deux : formulaire guidé, suivi prof en direct, infrastructure 100 % EN, modération possible.
Mizou
Outil généraliste puissant, mais hébergé hors EN, moins ancré dans les usages pédagogiques français.
