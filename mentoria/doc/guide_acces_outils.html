🛠️ Guide d'accès aux outils — Prérequis techniques

Ce qu'il faut obtenir pour déployer Experia et/ou Mentoria depuis zéro
⚡ Experia 🧭 Mentoria Enseignants & équipes techniques Académie de Corse — 2025-2026
Les deux outils reposent sur des services numériques de l'Éducation Nationale ou conformes RGPD. Aucun service commercial non encadré n'est utilisé. Ce guide explique comment obtenir l'accès à chacun d'eux. Certains accès nécessitent un compte académique (@ac-corse.fr ou équivalent) ; d'autres font l'objet d'une demande spécifique.
Vue d'ensemble — quel outil pour quel projet
Service Rôle Utilisé par Accès
Forge des Communs Hébergement des interfaces (GitLab Pages) Les deux Compte académique
Albert API IA souveraine — génération et correction Les deux Demande de clé API
Nuage national Stockage des données (WebDAV) Mentoria Compte académique
n8n incubateur Orchestration des flux IA Mentoria Demande projet incubateur
Cloudflare Workers Proxy IA + stockage sessions Experia Compte gratuit (email)

Forge des Communs Numériques

forge.aeif.fr
⚡ Experia 🧭 Mentoria
Instance GitLab de l'Éducation Nationale. Les fichiers HTML des interfaces (prof, élève, suivi) sont versionnés ici et déployés automatiquement en GitLab Pages via un fichier .gitlab-ci.yml. Chaque push sur la branche principale met à jour le site en ligne.
1
Aller sur forge.aeif.fr
2
Se connecter avec son compte académique (via SSO académique)
3
Créer un nouveau projet ou dupliquer (fork) le projet source
4
Activer GitLab Pages dans les paramètres du projet → l'URL publique est générée automatiquement
🤖

Albert API — IA souveraine

albert.api.etalab.gouv.fr
⚡ Experia 🧭 Mentoria
IA générative développée par Étalab / DINUM, hébergée sur des serveurs de l'État français. C'est le cœur des deux outils : génération d'activités pour Experia, réponses du bot pour Mentoria. Conforme RGPD par construction — les données ne quittent pas le territoire français.
1
Aller sur albert.api.etalab.gouv.fr
2
Créer un compte (email académique recommandé) ou se connecter via FranceConnect
3
Dans le tableau de bord, générer une clé API (token Bearer)
4
Noter le modèle disponible (ex. AgentPublic/llama3-instruct-8b) et l'URL de base de l'API
5
Insérer la clé dans la configuration du Worker (Experia) ou du workflow n8n (Mentoria)
⚠️ Quota : L'accès Albert API est soumis à quota. Pour un usage classe, le free tier est généralement suffisant. Des accès renforcés peuvent être demandés via la DINUM pour des projets institutionnels.
☁️

Nuage national

nuage.apps.education.fr
🧭 Mentoria uniquement
Instance Nextcloud hébergée par l'Éducation Nationale. Mentoria l'utilise comme base de stockage via le protocole WebDAV : configurations des bots, conversations élèves, logs. Toutes les données restent sur les serveurs de l'EN.
1
Aller sur nuage.apps.education.fr
2
Se connecter avec son compte académique
3
Créer un dossier dédié à Mentoria dans le Nuage
4
4
Générer un mot de passe d'application (Paramètres → Sécurité → Mots de passe d'application) pour l'accès WebDAV depuis n8n
5
Renseigner l'URL WebDAV et le mot de passe dans le workflow n8n
⚙️

n8n — Orchestrateur de flux

n8n.incubateur.education.gouv.fr
🧭 Mentoria uniquement
Instance n8n hébergée par l'incubateur de l'Éducation Nationale. N8n est un outil d'automatisation de flux (type Zapier, open source) qui relie les différents services entre eux : il reçoit les messages des élèves, interroge Albert API, sauvegarde les conversations sur le Nuage et renvoie les réponses.
1
Contacter l'équipe de l'incubateur via incubateur.education.gouv.fr pour demander un accès
2
Présenter le projet pédagogique (l'accès est réservé aux projets EN, pas à usage privé)
3
Une fois l'accès accordé, importer le fichier workflow JSON fourni dans le dépôt
4
Configurer les credentials (Albert API key, Nuage WebDAV) dans n8n
5
Activer le workflow et noter l'URL du webhook généré
⚠️ Usage éthique : Le n8n de l'incubateur est une ressource publique mutualisée, réservée aux projets pédagogiques de l'EN. Il ne doit pas être utilisé pour des activités privées ou commerciales.

Cloudflare Workers + KV

dash.cloudflare.com
⚡ Experia uniquement
Cloudflare Workers est un service de calcul en périphérie (edge computing) qui permet d'exécuter du code côté serveur sans gérer d'infrastructure. KV (Key-Value) est sa base de données associée. Dans Experia, il joue deux rôles : proxy sécurisé vers Albert API (pour éviter d'exposer la clé API dans le navigateur) et stockage des sessions (activités générées, réponses élèves, résultats).
🤔 Pourquoi Cloudflare et pas un service EN ? Experia a d'abord utilisé Firebase (Google), mais Firebase est bloqué dans de nombreux établissements scolaires. Cloudflare Workers, lui, passe partout. Par ailleurs, les services souverains de l'EN (Nuage, n8n) n'offrent pas nativement de proxy d'API ni de stockage clé-valeur temps réel adapté. Cloudflare a été retenu car il dispose d'un accord DPA RGPD, les données peuvent être localisées en Europe, et son free tier (100 000 requêtes/jour) est largement suffisant pour un usage classe.
1
Créer un compte gratuit sur cloudflare.com (email suffisant, pas de carte bancaire pour le free tier)
2
Dans le tableau de bord → Workers & Pages → créer un nouveau Worker
3
Coller le code du fichier experia_worker_v2.js fourni dans le dépôt
4
Créer un namespace KV (Workers → KV) et le lier au Worker
5
Définir les variables d'environnement : ALBERT_API_KEY, MASTER_SECRET
6
Déployer → noter l'URL du Worker (ex. experia-proxy.votrecompte.workers.dev)
7
Renseigner cette URL dans prof.html et eleve.html (variable WORKER_URL)
RGPD : Cloudflare est une entreprise américaine mais dispose d'un accord de traitement des données (DPA) conforme au RGPD. Les Workers peuvent être configurés pour traiter les données en Europe uniquement. Aucune donnée personnelle identifiable n'est stockée — les élèves utilisent des pseudonymes libres.
