Présentation

Qu'est-ce que Mentoria ?

Mentoria est un studio de création d'assistants IA pédagogiques personnalisés. En quelques minutes, un professeur configure un assistant sur mesure pour sa classe — avec ses propres instructions, son corpus documentaire et ses règles pédagogiques. L'élève y accède via un simple lien, sans compte, sans application.

🛠️

Le Studio (prof)

Interface de création : titre, personnalité, corpus, règles, avatar. Sauvegarde dans votre espace Nuage personnel.

💬

Le Compagnon (élève)

Interface de chat fluide — accès via un lien ou QR Code. Aucun compte, aucune installation. Fonctionne sur mobile.

🤖

L'IA Albert

Chaque échange passe par Albert, l'IA souveraine française hébergée par l'État (Etalab / DINUM). Aucune donnée envoyée hors UE.

📊

Suivi pédagogique

Le prof voit les échanges de ses élèves, peut exclure un élève, et supprime les logs à tout moment. Rétention 7 jours.

Ce que vous fournit votre référent numérique (Gilles Cormi) :
• Le lien vers le Studio Mentoria (à mettre en favori)
• Le lien vers le Compagnon (à partager aux élèves une fois votre assistant publié)

Ce que vous devez préparer vous-même (une seule fois) :
• Un dossier "mentoria" dans votre espace Nuage apps.education.fr
• Un mot de passe d'application Nuage (différent de votre mot de passe habituel)
• Une clé Albert API personnelle

La section Avant de commencer vous guide pas à pas pour tout préparer.
Différence avec Experia : dans Experia, le prof crée des évaluations avec des activités structurées (QCM, questions ouvertes…). Dans Mentoria, le prof crée un assistant conversationnel — l'élève dialogue librement avec l'IA autour d'un sujet ou d'un corpus de documents.
Fonctionnement

Guide Professeur — Préparation

Avant de commencer

Avant d'ouvrir le Studio pour la première fois, trois éléments doivent être préparés. Chacun ne se fait qu'une seule fois. Suivez les étapes dans l'ordre.

Les liens d'accès (Studio et Compagnon) sont fournis par Gilles Cormi, votre référent numérique. Contactez-le pour les obtenir si vous ne les avez pas encore.

Étape A — Créer un dossier "mentoria" dans Nuage

Mentoria stocke tous vos assistants et les échanges de vos élèves dans un dossier de votre espace Nuage. Il faut le créer manuellement une fois.

1

Connectez-vous sur nuage.apps.education.fr

Utilisez votre identifiant et mot de passe académique habituels (les mêmes que votre ENT).

2

Créez un nouveau dossier à la racine

Cliquez sur le bouton "+" ou "Nouveau" en haut de l'interface → "Nouveau dossier". Nommez-le exactement mentoria (tout en minuscules, sans espace, sans accent).

3

Vérifiez que le dossier est bien créé à la racine

Il doit apparaître directement dans votre espace, pas dans un sous-dossier. Mentoria cherchera précisément à cet emplacement.

Étape B — Créer un mot de passe d'application Nuage

Mentoria ne peut pas utiliser votre mot de passe ENT habituel pour accéder à Nuage — c'est une mesure de sécurité. Il faut créer un mot de passe d'application dédié, que vous utiliserez uniquement dans le Studio Mentoria.

1

Dans Nuage, cliquez sur votre avatar (en haut à droite)

Un menu apparaît. Cliquez sur "Paramètres".

2

Allez dans l'onglet "Sécurité"

Dans la page Paramètres, cliquez sur "Sécurité" dans le menu de gauche.

3

Créez un nouveau mot de passe d'application

Faites défiler jusqu'à la section "Mots de passe d'application". Dans le champ "Nom de l'application", tapez Mentoria. Cliquez "Créer un nouveau mot de passe d'application".

4

Copiez et conservez le mot de passe généré

Un mot de passe de 20 caractères s'affiche (ex : AbCdE-FgHiJ-KlMnO-PqRsT). Copiez-le immédiatement dans un endroit sûr (mémo, gestionnaire de mots de passe). Il n'est affiché qu'une seule fois.

Important : ce mot de passe d'application est différent de votre mot de passe ENT. Si vous le perdez, il suffit d'en créer un nouveau dans Nuage — vos fichiers ne sont pas affectés. Mettez simplement à jour le Studio Mentoria avec le nouveau mot de passe.

Étape C — Obtenir une clé Albert API

Albert est l'IA française qui alimente les assistants Mentoria. Pour l'utiliser, chaque professeur doit obtenir sa propre clé API — c'est gratuit pour les agents de l'Éducation nationale.

1

Ouvrez albert.api.etalab.gouv.fr

Ce site est géré par l'État français (DINUM / Etalab). Pas de compte commercial requis.

2

Cliquez "S'inscrire" ou "Créer un compte"

Utilisez votre adresse e-mail académique (ex : p.dupont@ac-corse.fr). Remplissez le formulaire d'inscription en indiquant votre établissement et votre rôle d'enseignant.

3

Validez votre compte par e-mail

Un lien de confirmation est envoyé à votre adresse académique. Cliquez dessus pour activer votre compte.

4

Générez une clé API

Une fois connecté, allez dans votre profil ou tableau de bord → section "Clés API" → cliquez "Créer une clé". Donnez-lui un nom (ex : Mentoria).

5

Copiez la clé et conservez-la

La clé commence par eyJ…. Copiez-la dans un endroit sûr. Vous la renseignerez dans la section "Sécurité & API" de chaque assistant que vous créez dans Mentoria.

Une fois ces trois étapes faites, vous avez tout ce qu'il faut pour utiliser Mentoria. Ces préparatifs ne sont à faire qu'une seule fois — ensuite, la création d'assistants et le partage aux élèves se font en quelques minutes.
Guide Professeur — Étape 1

Accéder au Studio

Le Studio Mentoria est une page web — aucune installation. Votre référent numérique vous communique le lien. Mettez-le en favori pour y revenir facilement.

1

Ouvrez le lien Studio transmis par votre référent

Le Studio s'ouvre dans votre navigateur habituel. Chrome ou Edge sont recommandés pour bénéficier de toutes les fonctionnalités (notamment le mode USB).

2

Choisissez votre mode de stockage

Un écran de démarrage vous propose trois modes. Le mode Nuage est recommandé pour une utilisation en classe (voir étape suivante).

3

Entrez vos identifiants et cliquez "Connexion"

Après connexion, votre bibliothèque d'assistants s'affiche dans la barre de gauche. Si c'est votre première connexion, elle est vide — normal.

Vos identifiants ne sont jamais envoyés à un serveur central. Ils restent dans votre navigateur et servent uniquement à accéder à votre propre espace Nuage. À chaque session, vous devrez les re-saisir (mesure de sécurité intentionnelle).

Guide Professeur — Étape 3

Créer un assistant

Un assistant Mentoria, c'est une personnalité IA construite autour de vos besoins pédagogiques. Il peut avoir un nom, un avatar, un domaine d'expertise, des règles de comportement et un corpus de documents.

1

Cliquez "+ Nouvel assistant" dans la barre de gauche

Un panneau de configuration s'ouvre au centre. Il est divisé en trois sections dépliables.

2

Section 1 — Identité & Présentation

Renseignez : le titre (visible dans la liste), le nom de l'assistant (celui que l'élève voit), le message d'accueil, l'âge cible (ex : "12-14 ans"), le thème ou matière, et optionnellement un avatar (image importée, max 128×128 px).

3

Section 2 — Pédagogie & Corpus

C'est le cœur de l'assistant. Rédigez les instructions IA (ex : "Tu es un professeur de mathématiques bienveillant…"). Cochez les règles pédagogiques prédéfinies (réponses concises, guidage socratique…). Collez votre corpus documentaire (textes de cours, extraits, fiches…). L'IA s'appuiera dessus pour répondre.

4

Configurer les boutons de réponse rapide (optionnel)

Ces boutons apparaissent sous la zone de chat de l'élève. Chaque bouton déclenche une demande ponctuelle prédéfinie (ex : "Expliquer ce concept autrement", "Donner un exemple"). Très utile pour guider les élèves moins à l'aise.

5

Section 3 — Sécurité & API

Entrez votre clé Albert API (commence par eyJ…). Si vous activez le suivi pédagogique, renseignez également votre identifiant Nuage, le mot de passe d'application et un identifiant de classe (ex : 3eA).

Poids de l'assistant : un indicateur en temps réel affiche le poids total du bot (corpus + avatar). Restez en dessous de 300 Ko pour que le lien élève reste confortable. Un corpus trop long allonge aussi le temps de réponse de l'IA.
Thème strict : si vous activez l'option "Thème strict", l'assistant refusera de répondre à toute question hors du sujet défini. Pratique pour les évaluations — moins adapté à un usage d'accompagnement ouvert.
Guide Professeur — Étape 4

Publier le lien élève

Une fois votre assistant configuré et sauvegardé, vous pouvez générer un lien élève. Ce lien contient toute la configuration de l'assistant encodée — l'élève n'a besoin de rien d'autre.

1

Sauvegardez d'abord l'assistant

Cliquez le bouton "Sauvegarder". L'assistant est écrit dans votre espace Nuage (ou USB). Un indicateur de synchronisation confirme la sauvegarde.

2

Cliquez "Publier"

Un lien est généré — c'est l'URL de la page compagnion.html avec la configuration de votre assistant encodée dans l'adresse.

3

Copiez et partagez le lien

Envoyez-le par ENT, Pronote, e-mail ou affichez le QR Code généré en classe. L'élève clique — aucun compte à créer, aucune installation.

4

L'élève entre son identifiant ENT

À l'ouverture du Compagnon, l'élève saisit son identifiant ENT (ou pseudonyme selon votre consigne). Cet identifiant sert à retrouver son historique de conversation et pour le suivi pédagogique.

Le lien est auto-suffisant : toute la configuration de l'assistant (instructions, corpus, règles…) est encodée dans l'URL elle-même. Si votre Nuage est indisponible, le lien fonctionne quand même pour les élèves (mais le suivi pédagogique est désactivé).
Lien long : si votre corpus est très volumineux ou si votre avatar est en haute résolution, l'URL peut devenir très longue. Mentoria compresse automatiquement la configuration, mais restez raisonnable. En cas d'URL trop longue, réduisez le corpus ou utilisez un avatar plus petit.

Ce que voit l'élève

L'élève arrive sur une page claire avec le nom et l'avatar de l'assistant, un message d'accueil, et un champ pour entrer son identifiant. Ensuite, il accède à une interface de chat fluide — il peut poser des questions, utiliser les boutons de réponse rapide, et récupérer l'historique de sa session précédente.

Reprise de session : si un élève revient sur le même lien avec le même identifiant, ses 40 derniers échanges sont rechargés automatiquement. Il reprend là où il s'était arrêté.
Guide Professeur — Étape 5

Suivre les élèves

Le panneau de Suivi pédagogique vous permet de consulter les échanges de vos élèves avec l'assistant, de modérer, et de purger les données.

Prérequis : le suivi n'est disponible que si vous avez activé la configuration de suivi dans la section "Sécurité & API" de l'assistant (identifiant Nuage + mot de passe app + identifiant de classe).
1

Cliquez "Suivi" dans le Studio

Un panneau s'ouvre (déplaçable et redimensionnable). Il liste tous les élèves qui ont interagi avec cet assistant pour la classe configurée.

2

Consultez les échanges par élève

Cliquez sur un identifiant élève pour voir l'historique complet de sa conversation — questions posées, réponses de l'IA, durée, nombre de messages.

3

Exclure un élève (bannissement)

Si un élève détourne l'assistant, cliquez "Bannir". La prochaine fois qu'il tentera de se connecter avec son identifiant, l'accès sera refusé. Le bannissement est réversible.

4

Supprimer les logs d'une classe

En bas du panneau, un bouton permet de supprimer tous les logs de la classe pour cet assistant. Utile entre deux groupes ou en fin d'année.

Rétention automatique : les logs sont automatiquement supprimés après 7 jours. Cette règle s'applique message par message — un échange de 6 jours reste visible, un échange de 8 jours disparaît. Aucune action manuelle requise.
RGPD : informez vos élèves que leurs échanges sont visibles par leur professeur pendant 7 jours. Ce message est affiché automatiquement sur la page d'accueil du Compagnon.
Guide Professeur — Étape 6

Gérer les assistants

Tous vos assistants sont listés dans la barre de gauche du Studio. Vous pouvez en créer autant que nécessaire — un par matière, par niveau, par séquence pédagogique.

1

Cliquer sur un assistant pour le modifier

La configuration se recharge dans le panneau central. Modifiez ce que vous souhaitez et sauvegardez.

2

Exporter un assistant (JSON)

Le bouton "Exporter JSON" télécharge la configuration complète de l'assistant sélectionné. Utile pour sauvegarder, partager avec un collègue, ou réimporter plus tard.

3

Importer un assistant (JSON)

Le bouton "Importer JSON" charge un assistant depuis un fichier exporté (le vôtre ou celui d'un collègue). Il apparaît dans votre bibliothèque avec un nouvel identifiant.

4

Supprimer un assistant

Une icône de suppression est disponible sur chaque assistant dans la liste. La suppression retire le fichier de votre Nuage (ou USB) et met à jour le manifeste.

Partage entre collègues : exportez un assistant en JSON et envoyez le fichier à un collègue. Il l'importe dans son propre Studio. Chacun peut ensuite le personnaliser indépendamment — aucune dépendance entre les comptes.

Évaluation automatique de la qualité de la discussion

Aujourd'hui, le panneau Suivi montre les échanges élèves mais ne les évalue pas. L'évolution prévue : Albert analyse l'historique complet de la conversation et produit une grille d'évaluation sur 10, visible dans le Suivi pédagogique.

L'intérêt pédagogique est distinct d'Experia — ce n'est pas une évaluation des réponses correctes, mais de la démarche intellectuelle : un élève curieux qui pose beaucoup de questions et approfondit ses échanges est valorisé, même s'il ne maîtrise pas encore tout le contenu.

Notice Technique

Déploiement du Worker Cloudflare

Le Worker Cloudflare est un proxy CORS léger — il ne stocke rien, ne logge rien. Son unique rôle est de permettre aux navigateurs des élèves d'appeler l'API Albert sans blocage CORS.

Créer le Worker

1

dash.cloudflare.com → Compute → Workers & Pages → Create

Choisissez "Create Worker", nommez-le mentoria-proxy, cliquez "Deploy".

2

Coller le code de worker.js

Cliquez "Edit code", sélectionnez tout (Ctrl+A), collez le contenu du fichier worker.js, puis cliquez "Deploy".

// URL du Worker après déploiement https://mentoria-proxy.gilles-cormi.workers.dev // Cette URL est codée en dur dans compagnion.html // Si vous créez votre propre Worker, mettez à jour cette valeur

Ce que fait le Worker

Le Worker reçoit les requêtes de compagnion.html avec la clé Albert du prof encodée, puis les transmet à albert.api.etalab.gouv.fr. Il ajoute les en-têtes CORS nécessaires pour que les navigateurs acceptent la réponse.

Plan gratuit Cloudflare : 100 000 requêtes/jour. Chaque échange élève = 1 requête. Largement suffisant pour une phase test avec 20-30 profs et des classes de 30 élèves.

Mettre à jour le Worker

Workers & Pages → mentoria-proxy → Edit code → Ctrl+A → Coller le nouveau worker.js → Deploy. Les sessions en cours ne sont pas interrompues.

Workflows n8n

Importer le workflow dans n8n

1

Connexion à n8n.incubateur.education.gouv.fr

Demandez un accès à l'équipe n8n de l'incubateur, ou déployez votre propre instance n8n.

2

Importer mentoria_storage.json

Dans n8n : menu hamburger → Import workflow → sélectionner le fichier mentoria_storage.json. Le workflow apparaît dans votre espace.

3

Activer le webhook

Cliquez sur le nœud "Webhook" dans le workflow → copiez l'URL de production. C'est cette URL qui doit être renseignée dans index.html comme STORAGE_PROXY_URL.

En cas de perte du mot de passe d'application : supprimez-le dans Nuage et créez-en un nouveau. Les fichiers bots existants ne sont pas affectés. Mettez à jour le mot de passe dans le Studio Mentoria.

Notice Technique

Maintenance

Mettre à jour les pages HTML (Studio et Compagnon)

1

Modifier index.html ou compagnion.html dans le dossier de travail

Les fichiers sources sont dans mentoria/ (dossier de développement).

2

Copier vers deploy_site/mentoria/ et lancer firebase deploy

Les commandes à exécuter depuis le dossier frojet/ :

# Copier les fichiers mis à jour cp mentoria/index.html deploy_site/mentoria/index.html cp mentoria/compagnion.html deploy_site/mentoria/compagnion.html # Déployer sur Firebase (depuis frojet/) firebase deploy

Mettre à jour le Worker Cloudflare

1

dash.cloudflare.com → Workers & Pages → mentoria-proxy → Edit code

Ctrl+A → Coller le nouveau worker.js → Deploy. Aucune session en cours n'est interrompue.

Supprimer les données d'un prof (droit à l'effacement)

Demandez au prof de supprimer lui-même le dossier mentoria/ dans son espace Nuage, ou d'en supprimer les fichiers individuellement. Vous n'avez pas accès à cet espace en tant que référent.

Réinitialiser le mot de passe d'application Nuage d'un prof

Le prof va dans Nuage → Paramètres → Sécurité → Supprime l'ancien mot de passe "Mentoria" → Crée un nouveau → Renseigne le nouveau dans le Studio. Ses fichiers bots ne sont pas affectés.

Notice Technique

Conformité RGPD

Mentoria a été conçu avec la souveraineté des données comme contrainte principale. L'architecture "stockage chez le prof" élimine toute base de données centrale.

🏠

Données chez le prof

Chaque prof stocke ses bots et les logs de ses élèves dans son propre espace Nuage. Le référent Mentoria n'a aucun accès à ces données.

👤

Élèves pseudonymes

L'élève entre son identifiant ENT ou un pseudonyme — aucun e-mail, aucune donnée biographique. Le prof voit cet identifiant, pas une identité complète.

🕐

Logs limités à 7 jours

Les échanges élèves sont automatiquement filtrés après 7 jours. Le prof peut aussi supprimer manuellement à tout moment.

🇫🇷

IA souveraine

Albert API est hébergée par le DINUM (État français). Les réponses des élèves n'alimentent pas des modèles commerciaux étrangers.

Transparence vis-à-vis des élèves

La page d'accueil du Compagnon affiche automatiquement ce message aux élèves : "Vos échanges sont visibles par votre enseignant pendant 7 jours à des fins pédagogiques." Aucune action supplémentaire requise du prof.

Conclusion : Mentoria est conforme aux principes RGPD de minimisation des données, de pseudonymisation, de limitation de la durée de conservation et de souveraineté des traitements IA. Les données des élèves restent dans l'espace académique du professeur — jamais dans un cloud commercial.
Point d'attention : la clé Albert et le mot de passe d'application Nuage sont encodés dans le fichier JSON du bot, qui lui-même est encodé dans l'URL publiée. Recommandez aux profs de ne pas partager ce lien sur des plateformes publiques indexées.
